Bake a wide range of texture maps, preview them before baking, inspect the baked textures, export the baked textures and objects, apply the baked textures, create a baked version of the objects and materials. Bake UDIM and atlas textures, apply anti-aliasing, channel pack the textures. Generate new UVs, choose a target UV, remove unused UV maps, bake from source to target objects, generate cage objects, visualize extrusion, create and use your custom presets, and choose your workspace.

WORKFLOW

Set up a bake list for the textures, select the objects in the viewport you want to bake, then start the bake, and follow the progress in the bake queue! Create a baked version of the objects with applying the baked textures to them. Export the baked textures and objects with your own export format and settings.

Designed for both beginner and advanced Blender users, with barely anything needed to change to bake and use it, but with high custimibility options. A provided documentation that guides you through each steps, with great details and examples.

MATERIAL SUPPORT

Bake both the very simple materials, the same way as the complex material setups with multiple shaders mixed and added together. Node groups can also be used both simple or nested node groups. Objects can also have multiple materials assigned to them.

Node Setup Example

Use shader nodes other than the Principled BSDF. While the shader texture maps are designed around the Principled BSDF, other shader nodes are also supported to bake, some are partially while some are fully, depending on the shader's input sockets.

Fully & Partially Supported Examples

When a shader node has matching input sockets with the Principled BSDF node, the matching sockets then can be baked iwth the the same shader texture map as the Principled BSDF node.

BAKED VERSION

Convert the baked objects and their textures into a baked and final version, with a copy of the original object with a new final material assigned to it, and the baked textures automatically applied to it, with the texture nodes aligned to each other and connected to their corresponding shader sockets.

Automatic Texture Setup

This process will result a complete conversion from the original unbaked object and its materials, to a baked version of it.

UDIM

Auto Bake offers a full UDIM texture support with a dedicated UDIM. In the UDIM mode there is also a UDIM Tiles list which holds the tiles and their scales. The tiles can either be set one by one or imported from an existing UDIM texture.

UDIM List
UDIM Labels

For the UDIM tiles labels can be enabled, that then can be freely edited, and when baking the textures, these labels will be assigned to the UDIM tiles of the baked textures.

UV TOOLS

Auto Bake is suited with useful UV tools, which one of them allows to bake to a different UV map. Baking to a different UV map opens the possibility to use another or multiple UV maps for the materials, or to rebake textures from one UV to another. The target UV map can either be the default that is used for rendering, the active UV map, or one specified by a set name.

Auto Bake can also generate UV maps using Blender's Smart UV Project. A new UV map is generated when the set target UV is not found, or if the object has no UV map at all.

Target UV
Smart UV Project

The unused UV maps can also be removed from the object after the bake, only keeping the one the textures was baked to.

TEXTURE TOOLS

Channel packing allows to bake up to three white-and-black textures (like the Roughness, Metallic, or Alpha) as a single RGB, combining them into one texture instead of three separate. You can customize what texture map to have on which RGB channel, or which to left empty.

{img of how channel packing works} {img of how the channels can be selected}

Texture sharing / atlas bake is another useful feature of Auto Bake, which allows to bake multiple objects onto the same textures, just by enabling a single option.

{mp4 of the bake process where on the left the objects are selected and on the right the base color is baked from one object then the next and the next}

The quality of the textures depend a lot on the used sampling settings when baking, but not for all the textures. Some texture maps do not require high settings at all, and only the bare minimum without any loss of quality. For this reason Auto Bake uses a split render sampling method, where two different render settings are used, one for the textures that not sample dependent, and the other that is. Of course this can also be disabled an a single setting can be used for all.

Sampling Low
Sampling High

Different texture maps need to be used in different color-spaces, which by default should be changed one by one but Auto Bake will automatically set the correct one by default. Of course when you want to use a different color-space for a texture type, it can be changed with a single option.

Keeping the textures and preventing them from being lost can be done by either exporting them or the other method is packing them into the .blend file itself. The options are granted for both, or you can also assign a fake user automatically to the fake user.

Anti-aliasing can be enabled to apply to the baked textures. You can choose between to option methods, the upscaled where the texture is baked in a set higher resolution then scaled down, or the downscaled where the textures after the bake are scaled down and back up.

The default color of the textures can also be edited to any custom color, or also the alpha channel generation can also be enabled. These parts are only visible on the textures where nothing is baked, but can be used with low margin for chroma keying or other.
