It can also export textures, create a baked version of the objectss with automatic texture setup, export the baked objects, supports advanced materials, and a lot more!
WORKFLOW

Set up a bake list for the textures you want to bake, then simply select the objects in the viewport you want to bake, and the bake is ready to be started! After the bakes are finished, a baked version of the objects are created with the baked textures automatically applied to them. The baked textures and objects can then be exported with a single button with your custom export settings.

EASE OF USE

Auto Bake is designed for both beginners and advanced users in mind. The panels and their content are thoughtly organized, the buttons and options are enchanced with tooltips to help knowing how to use them. Text alerts are added for the options when something is wrongly set (example an invalid filepath). The documentation explains the buttons and options in great details and with examples.

A wide variety of textures can be baked with Auto Bake, including shader sockets, multiresolution modifier data, Blender's default bake passes, and a set of miscellaneous bake types for channel packing textures, baking pointiness map of the object, or baking bevel normal, and more!

PREVIEW TEXTURES

You also have to option the preview the textures maps before baking them. The preview can be enabled with a single button for all the selected objects. The preview is avialable for all the shader and miscellaneous bake types.

Texture Preview

Certain texture maps also have adjustable properties, like the channel packing which for you can set to what to bake onto each channels, or the roughness which can be inverted into a smoothness map for unity, or controlling the ambient occlusion, or adjusting the pointiness map and more!

Real-time Preview Update
MATERIAL SUPPORT

Auto Bake is designed both for the very basic single material per objects and for the advanced setups with multiple materials per object. The materials can also vary from a single shader node to the complex node setups with mixing multiple shader nodes and using node groups.

Node Setup Example

Although the texture maps are designed around the Principled BSDF node, it also supports all the other shaders, some are to a fully and some to partially extent, depending on the shader's input sockets. If a socket has a matching pair with one of the Principled BSDF's socket, then it can be baked using the same texture map as for the Principled BSDF node.

BAKED VERSION

By enabling a single option, Auto Bake will create a final material for the baked textures, and automatically set them up, connecting them to their corresponding input socket of the shader node, with the nodes automatically position and aligned.

Automatic Texture Setup

With another option enabled, Auto Bake will not only set up the materials and textures, but it will also create a baked version of all the baked objects, applying their own final materials. Additionally, then the original object can be completely replaced with the baked object using the confirm button.

EXPORT

Auto Bake gives the option to export either or both the baked textures and the objects with a single button, with your own custom export settings. You can choose different file format both for the textures and objects.

Texture & Object Export

Options are also provided for folder control. Use of master folders can be enabled to place both the exported object and its textures in. Subfolders can also be enabled to organize the textures by type.

UDIM

Auto Bake offers a full UDIM texture support with a dedicated UDIM. In the UDIM mode there is also a UDIM Tiles list which holds the tiles and their scales. The tiles can either be set one by one or imported from an existing UDIM texture.

UDIM List
UDIM Labels

For the UDIM tiles labels can be enabled, that then can be freely edited, and when baking the textures, these labels will be assigned to the UDIM tiles of the baked textures.

UV TOOLS

Auto Bake is suited with useful UV tools, which one of them allows to bake to a different UV map. Baking to a different UV map opens the possibility to use another or multiple UV maps for the materials, or to rebake textures from one UV to another. The target UV map can either be the default that is used for rendering, the active UV map, or one specified by a set name.

Auto Bake can also generate UV maps using Blender's Smart UV Project. A new UV map is generated when the set target UV is not found, or if the object has no UV map at all.

Target UV
Smart UV Project

The unused UV maps can also be removed from the object after the bake, only keeping the one the textures was baked to.

TEXTURE TOOLS

Channel packing allows to bake up to three white-and-black textures (like the Roughness, Metallic, or Alpha) as a single RGB, combining them into one texture instead of three separate. You can customize what texture map to have on which RGB channel, or which to left empty.

{img of how channel packing works} {img of how the channels can be selected}

Texture sharing / atlas bake is another useful feature of Auto Bake, which allows to bake multiple objects onto the same textures, just by enabling a single option.

{mp4 of the bake process where on the left the objects are selected and on the right the base color is baked from one object then the next and the next}

The quality of the textures depend a lot on the used sampling settings when baking, but not for all the textures. Some texture maps do not require high settings at all, and only the bare minimum without any loss of quality. For this reason Auto Bake uses a split render sampling method, where two different render settings are used, one for the textures that not sample dependent, and the other that is. Of course this can also be disabled an a single setting can be used for all.

Sampling Low
Sampling High

Different texture maps need to be used in different color-spaces, which by default should be changed one by one but Auto Bake will automatically set the correct one by default. Of course when you want to use a different color-space for a texture type, it can be changed with a single option.

Keeping the textures and preventing them from being lost can be done by either exporting them or the other method is packing them into the .blend file itself. The options are granted for both, or you can also assign a fake user automatically to the fake user.

Anti-aliasing can be enabled to apply to the baked textures. You can choose between to option methods, the upscaled where the texture is baked in a set higher resolution then scaled down, or the downscaled where the textures after the bake are scaled down and back up.

The default color of the textures can also be edited to any custom color, or also the alpha channel generation can also be enabled. These parts are only visible on the textures where nothing is baked, but can be used with low margin for chroma keying or other.
